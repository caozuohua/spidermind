# SpiderMind 开发计划

## 开发原则

- 每个模块独立开发、测试验证
- 验证通过后立即提交并推送 GitHub
- 测试记录保存在 `devlog/` 目录
- 按依赖顺序开发（底层模块优先）

## 模块依赖图

```
Phase 1: 基础层（无外部依赖）
  M1 数据模型 ←── 所有模块依赖
  M2 配置系统
  M3 插件注册系统

Phase 2: 核心组件（依赖 Phase 1）
  M4 HTTP 下载器
  M5 CSS/XPath 提取器
  M6 数据管道
  M7 存储后端
  M8 调度器

Phase 3: 集成层（依赖 Phase 2）
  M9 引擎引擎（组装所有组件）
  M10 AI 提取器
  M11 CLI 命令行

Phase 4: 完善
  M12 文档与示例
  M13 集成测试与发布
```
